This Python script is designed to preprocess an image dataset, specifically for the "FantasyID Dataset," by creating an LMDB (Lightning Memory-Mapped Database) from image and annotation pairs.

**Key functionalities and changes include:**

*   **Sample Discovery:** The `find_samples` function recursively searches a specified directory for `.jpg`/`.jpeg` images and their corresponding `.json` annotation files, pairing them based on their stem.
*   **Annotation Processing:** The `load_regions` function reads region data from JSON files. The `build_mask_from_bboxes` function then creates a binary (0/255) mask from these annotations, specifically targeting "altered" rectangular regions.
*   **Image and Mask Resizing/Padding:** A core function, `resize_and_pad_to_512`, resizes images and their generated masks to a uniform `512x512` (or a configurable `out_size`) while preserving the aspect ratio. It uses different interpolation methods for images (bilinear/area) and masks (nearest-neighbor) to maintain mask integrity, and then pads them to the target square dimension. Metadata about the original size, scale, and padding is captured.
*   **LMDB Creation:** The `write_lmdb` function orchestrates the dataset conversion.
    *   It first scans and prints statistics about the original image sizes (min/max height/width).
    *   It then opens an LMDB environment, iterating through each image-annotation pair.
    *   For each pair, it reads the image, generates the mask, resizes/pads them, and encodes the processed image to JPG bytes and the mask to PNG bytes (for lossless storage of binary masks).
    *   These encoded bytes are stored in the LMDB with sequential keys like `image-000000000` and `label-000000000`.
    *   It also tracks and logs worst-case padding statistics to understand data distortion.
    *   Finally, the total number of samples processed is stored under the `num-samples` key.
*   **Command-Line Interface:** The `main` function uses `argparse` to allow users to specify the `dataset_root`, `out_root`, specific `splits` (e.g., "train", "val", "test"), `map_size_gb` for the LMDB, and the `out_size` for the images/masks.
*   **Dependencies:** The script relies on `json`, `argparse`, `pathlib`, `lmdb`, `numpy`, and `cv2` (OpenCV) for its operations.
