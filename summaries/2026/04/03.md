**Key functionalities provided in this file include:**

All models are configured with `class_weight="balanced"` or similar settings to handle potential class imbalance.
*   **`get_vectorizer_grid()`**: Defines four common text vectorization strategies: "Unigram Count", "Unigram TF-IDF", "Bigram Count", and "Bigram TF-IDF". Each `CountVectorizer` or `TfidfVectorizer` is consistently configured with `max_features=20_000` and `min_df=2`.
*   **`get_model_grid()`**: Provides an expanded set of classification models for a more exhaustive baseline search, including the models from `get_models()` along with `SVC` (RBF kernel) and a `VotingClassifier` ensemble (combining Logistic Regression, Complement Naive Bayes, and Calibrated Linear SVM using hard voting).
*   **`run_baseline_grid()`**: This is the core function, orchestrating an exhaustive grid search across all combinations of vectorizers from `get_vectorizer_grid()` and models from `get_model_grid()`. For each combination, it performs `k`-fold stratified cross-validation on the training data, then trains the model on the full training set and evaluates it on a hold-out test set. It calculates and logs various performance metrics including CV F1 (mean and std), Test Accuracy, Precision, Recall, and F1 (all macro-averaged). The function includes error handling for models requiring non-negative features and uses `sklearn.base.clone()` to prevent state corruption across runs. Results are sorted by CV F1 score, the top combinations are printed, and the full results can optionally be saved to a CSV file.
